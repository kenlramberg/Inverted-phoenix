## What We're Building

A digital wishing well where:
- People ask for what they need
- Others contribute what they can give
- The system coordinates fulfillment
- No ads. No paid ranking. No extraction.
- Pure human goodwill, scaled.

**The tech stack:**
- React 19 (Frontend)
- FastAPI (Backend)
- MongoDB (Database)
- AI-powered search (GPT-5.2)
- Progressive revelation UI
- Ethical matching algorithms

## What We're Asking

We need developers to improve specific parts of the codebase. **Small, focused contributions.** Not months of work—hours or days.

**Example tasks:**

4. Create an email notification system
5. Enhance the progressive revelation UI
6. Build automated testing for ethics engine
7. Improve mobile responsiveness

9. Optimize database queries
10. Build analytics dashboard (privacy-preserving)

Each task is **clearly defined** with specifications, acceptance criteria, and estimated time.

### Attachment: Sample Coding Tasks

**TASK 001: Optimize Contribution Matching Algorithm**
- **File:** `/app/backend/server.py` (match_offers_to_request function)
- **Current issue:** Basic keyword matching, low accuracy
- **Improvement needed:** Implement semantic matching using embeddings
- **Estimated time:** 4-6 hours
- **Skills:** Python, NLP, vector similarity
- **Impact:** Better matching = more fulfilled wishes

**TASK 002: Build Email Notification System**
- **Files:** New file `/app/backend/notifications.py`
- **Requirement:** Send emails when requests are matched/fulfilled
- **Tech:** Python email library or SendGrid integration
- **Estimated time:** 3-4 hours
- **Skills:** Python, email APIs
- **Impact:** Users know when their wishes are being worked on

**TASK 003: Mobile UI Improvements**
- **Files:** `/app/frontend/src/pages/*.js`
- **Issue:** Some pages not fully responsive on mobile
- **Improvement:** Fix responsive breakpoints, improve touch targets
- **Estimated time:** 2-3 hours
- **Skills:** React, Tailwind CSS, responsive design
- **Impact:** Better mobile experience

**TASK 004: Add Accessibility Features**
- **Files:** All frontend components
- **Requirement:** WCAG 2.1 AA compliance
- **Tasks:** Keyboard navigation, screen reader support, ARIA labels
- **Estimated time:** 5-6 hours
- **Skills:** React, accessibility best practices
- **Impact:** Platform accessible to everyone

**TASK 005: Build Reputation Scoring Engine**
- **Files:** New `/app/backend/reputation.py`
- **Requirement:** Calculate user reputation based on contributions

- **Estimated time:** 6-8 hours
- **Skills:** Python, algorithm design
- **Impact:** Gamification drives more contribution

**TASK 006: Database Query Optimization**
- **Files:** `/app/backend/server.py`

- **Estimated time:** 3-4 hours

- **Impact:** Faster platform as it scales

**TASK 007: Automated Testing Suite**
- **Files:** New `/app/tests/` directory

- **Estimated time:** 6-8 hours

**TASK 008: Analytics Dashboard (Privacy-Preserving)**
- **Files:** New page `/app/frontend/src/pages/Analytics.js`
- **Requirement:** Show platform metrics without tracking individuals
- **Metrics:** Total requests, fulfillment rate, contribution types
- **Estimated time:** 4-6 hours
- **Skills:** React, data visualization
- **Impact:** Transparency into how well the system works

**TASK 009: Progressive Revelation Enhancement**
- **Files:** `/app/frontend/src/pages/Journey.js`
